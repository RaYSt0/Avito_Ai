# Личный кабинет продавца с AI-ассистентом

## Структура репозитория

- `frontend` — React 18+ / TypeScript / Vite, интерфейс и вызовы API.
- `backend` — Node.js + Express, мок API в формате задания (`GET/PUT /items`).

## Что нужно установить

- **Node.js** версии 20 или новее (`node -v`).
- **npm** (идёт вместе с Node).
- **Ollama** — для кнопок «Придумать/Улучшить описание» и «Узнать рыночную цену» ([ollama.com](https://ollama.com/)).

### Модель для Ollama

В проекте ожидается модель **`llama3`** (подойдёт тег `llama3:latest`).

```bash
ollama pull llama3
```

Убедитесь, что демон запущен (часто достаточно держать открытым терминал с `ollama serve`, либо Ollama уже в фоне на macOS).

Проверка:

```bash
curl -s http://localhost:11434/api/tags
```

Первый запрос к модели после простоя может занять **десятки секунд** — это нормально.

## Запуск проекта

Нужны **два терминала**: backend и frontend. Ollama должна быть доступна на `http://localhost:11434`.

### 1. Backend (порт 8080)

```bash
cd backend
npm install
npm start
```

Должна появиться строка вроде `Backend started on port 8080`.

Другой порт (опционально):

```bash
PORT=8081 npm start
```

Если сменили порт, во `frontend` нужно поправить базовый URL API в `frontend/src/api.ts` (сейчас `http://localhost:8080`).

### 2. Frontend (по умолчанию порт 5173)

```bash
cd frontend
npm install
npm run dev
```

Откройте в браузере: **http://localhost:5173/** — произойдёт редирект на **http://localhost:5173/ads**.

### 3. Быстрая проверка, что всё живое

- Список объявлений: [http://localhost:8080/items?limit=10](http://localhost:8080/items?limit=10)
- Страница приложения: [http://localhost:5173/ads](http://localhost:5173/ads)

## AI (Ollama)

Фронтенд обращается к **`POST http://localhost:11434/api/generate`** с моделью `llama3`.

- Если Ollama не запущена или модели нет — описание и цена всё равно подставятся **заглушкой** (fallback), приложение не падает.
- Когда Ollama и модель на месте — ответы будут от настоящей LLM.

## Маршруты приложения

| URL | Назначение |
|-----|------------|
| `/ads` | Список объявлений |
| `/ads/:id` | Просмотр карточки |
| `/ads/:id/edit` | Редактирование и AI-кнопки |

## Принятые решения (кратко)

- Backend вынесен в отдельную папку `backend`, чтобы структура совпадала с ожидаемой разметкой репозитория; контракт API согласован с текстом задания.
- Загрузка черновика формы редактирования хранится в `localStorage` под ключом `ad_draft_<id>`.
- Цвета и вёрстка упрощены относительно Figma; при необходимости можно довести до пиксель-перфекта по экспорту из макета.

## Сборка production-версии фронтенда

```bash
cd frontend
npm run build
```

Статика появится в `frontend/dist`.
